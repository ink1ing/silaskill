# Operational Notes

## 为什么叫 “without gateway”

这里的 `gateway` 指 Home Hub / 家庭中枢，例如 Apple TV、HomePod 或常驻可作为中枢的设备。

没有中枢时，通常意味着：

- 你依赖本地网络
- 你依赖当前这台 Mac
- 你无法把逻辑交给 HomeKit 云端自动化

## 稳定性建议

- 让 `Home.app` 保持可正常打开
- 定时任务前尽量保证 Mac 处于唤醒状态
- 尽量不要让 Home 长时间停留在与脚本预期不一致的页面
- 升级 macOS 后立即跑一次开关回归测试

## 调试建议

- 先单独执行 Swift 控制脚本，不要一上来就查 `launchd`
- 如果失败，优先判断是“找不到窗口”还是“找到控件但状态未切换”
- 记录以下信息：
  - macOS 版本
  - Home 当前页面结构
  - 目标控件的 `AXRole` / `AXSubrole`
  - 切换前后读取到的 `On/Off` 值

## 推荐日志内容

- 执行时间
- 目标设备名
- 目标状态
- 切换前状态
- 切换后状态
- 退出码
- 异常消息

## 安全与权限

- 这是 GUI 自动化，不是官方 HomeKit API 授权链路
- 不要把控制脚本暴露给不受信任用户
- 如果用 `launchd`，确认脚本路径、日志路径、权限归属都属于当前用户

## 验证清单

每次系统升级后至少验证以下四项：

1. `Home.app` 能被正常激活
2. 目标设备名称仍能被脚本识别
3. 开关控件仍能被点击
4. 状态回读仍能正确得到 `On/Off`

## 建议的下一步

- 如果你只控制一个设备，直接从布局一开始
- 如果你已经有固定开关时间，采用布局二
- 如果后续要支持多个设备，建议尽早把“设备配置”和“控制逻辑”拆开
